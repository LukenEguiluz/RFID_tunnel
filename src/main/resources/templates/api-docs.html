📡 APIs de Lectores

GET Obtener todos los lectores

/api/readers
Obtiene la lista de todos los lectores configurados

GET Obtener un lector específico

/api/readers/{id}
Obtiene la información de un lector específico
Parámetros:
- id: ID del lector

GET Estado de un lector

/api/readers/{id}/status
Obtiene el estado actual de un lector (conectado, leyendo, etc.)

POST Iniciar lectura

/api/readers/{id}/start
Inicia la lectura de tags en un lector

POST Detener lectura

/api/readers/{id}/stop
Detiene la lectura de tags en un lector

POST Reset de conexión

/api/readers/{id}/reset
Reinicia la conexión del lector (desconecta y reconecta después de 2 segundos)
Respuesta: 
{
  "message": "Reader reset and reconnecting",
  "readerId": "lector-1"
}

POST Reboot completo

/api/readers/{id}/reboot
Ejecuta un reboot completo del lector (desconecta y reconecta después de 5 segundos)
Respuesta: 
{
  "message": "Reader reboot initiated, will reconnect in 5 seconds",
  "readerId": "lector-1"
}

POST Reset de antenas (por lector)

/api/readers/{id}/antennas/reset
Reinicia solo la configuración de antenas del lector sin desconectarlo
Respuesta: 
{
  "message": "Antennas configuration reset",
  "readerId": "lector-1"
}

👥 APIs de Grupos de Lectores

Permiten listar, crear y gestionar grupos de lectores. Ver documentación completa en API_GRUPOS_LECTORES.md.

GET Listar todos los grupos

/api/groups
Obtiene la lista de todos los grupos de lectores configurados.

GET Obtener un grupo por ID

/api/groups/{id}
Obtiene un grupo con sus lectores.

POST Crear grupo

/api/groups
Body JSON: id, name, description, readerIds (array), enabled.

PUT Actualizar grupo

/api/groups/{id}
Body JSON: name, description, readerIds, enabled.

GET Estadísticas del grupo

/api/groups/{id}/stats
totalReaders, enabledReaders, connectedReaders.

📖 Sesiones de lectura

Iniciar/detener lectura por lector o por grupo. Si recibes 409 Conflict, usa Reset forzado y vuelve a iniciar.

POST Iniciar sesión

/api/sessions/start
Body: {"readerId": "id"} o {"groupId": "id"}. Devuelve sessionId.

POST Reset forzado de sesión (evitar 409)

/api/sessions/force-reset
Detiene sesiones activas para poder iniciar de nuevo. Body: {"readerId": "id"} para un lector, {"groupId": "id"} para un grupo, o {} para todo.

GET Sesiones activas

/api/sessions/active
Lista de sesiones activas (lector o grupo).

📡 APIs de Antenas

GET Listar todas las antenas

/api/antennas
Obtiene la lista de todas las antenas configuradas

GET Obtener antena por ID

/api/antennas/{id}
Obtiene la información de una antena específica

POST Reset de antena por ID

/api/antennas/{antennaId}/reset
Reinicia la configuración de antenas del lector al que pertenece esta antena. Usa el ID de la antena (ej: reader-1-antenna-1)
Respuesta: 
{
  "message": "Antenna configuration reset",
  "antennaId": "reader-1-antenna-1",
  "readerId": "reader-1"
}

PUT Actualizar antena

/api/antennas/{id}
Actualiza antena (enabled, name, txPowerDbm, rxSensitivityDbm). Después aplicar cambios: POST /api/antennas/{id}/reset
Body JSON: enabled, name, txPowerDbm, rxSensitivityDbm

⚡ APIs de Eventos en Tiempo Real

GET Eventos en tiempo real (SSE)

/api/realtime/events
Stream de eventos en tiempo real usando Server-Sent Events (SSE). Incluye detecciones de tags y estadísticas.
Parámetros opcionales (query):
- readerId: Filtrar eventos por lector específico
- antenna: Filtrar eventos por antena específica
Ejemplo de uso: 
const eventSource = new EventSource('/api/realtime/events?readerId=lector-1');
eventSource.onmessage = (event) => {
    const data = JSON.parse(event.data);
    console.log('Tag detectado:', data);
};
